# Шаблон task-файлу для нового курсу

Зберегти як `task_YYYY-MM-DD.md` у корені проєкту.

```markdown
# Завдання: створити практичний курс [ТЕМА]

## Мета

[Для кого курс, які реальні задачі вирішує]

## Вхідні дані

[Реальне середовище користувача: shell, доступні команди, інструменти]

## Основний принцип

Не всі команди — лише 30–50 ключових за сценаріями:

1. [Сценарій 1]
2. [Сценарій 2]
…

## Структура курсу

\`\`\`text
<course-slug>/
├── README.md
├── 01-<topic>.md
…
└── cheatsheet.md
\`\`\`

## Вимоги до кожного уроку

1. Коротке пояснення
2. Список команд з: що робить, коли, приклад, результат, помилки, ризик
3. Практичне завдання
4. Міні-чекліст

## Урок 1 — [назва]

Команди:
\`\`\`bash
command1
command2
\`\`\`

(повторити для уроків 2–10)

## Cheatsheet

Розділи: Navigation, Files, …

## Стиль

- Мова: українська
- Практично, без академічної теорії

## Фінальний результат

- Markdown-курс у `<course-slug>/`
- HTML-тренажер (як git-course-emulator)
- GitHub репозиторій + GitHub Pages
```

## Як агент використовує task

1. Прочитати `task_*.md` повністю
2. Витягнути список команд по уроках → MODULES + UK_HINTS
3. Витягнути сценарії → модулі тренажера
4. Згенерувати markdown за `lesson-format.md`
5. Зібрати HTML за `emulator-patterns.md`
6. Задеплоїти за `github-deploy.md`
