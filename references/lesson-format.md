# Формат Markdown-уроків

## Мова і стиль

- Українська мова (якщо task не вказує інше)
- Практично, коротко, по-інженерному — без академічної теорії
- Фокус на 30–50 ключових команд, не на сотні системних

## Обов'язкова фраза

Включити в `README.md` курсу та в кожен урок (на початку або в блоці «Принцип»):

```text
Головна мета — не вивчити всі команди, а навчитися швидко знаходити потрібну команду, розуміти її ризик і застосовувати її в реальному сценарії.
```

## Структура кожного уроку

```markdown
# N. Назва уроку

[Коротке пояснення теми — 2-4 речення]

## Команди

### `command args`

| | |
|---|---|
| **Що робить** | … |
| **Коли використовувати** | … |
| **Приклад** | `command args` |
| **Типовий результат** | … |
| **Типові помилки** | … |
| **Ризик** | низький / середній / високий |

(повторити для кожної команди)

## Практичне завдання

1. …
2. …
3. …

## Міні-чекліст

- [ ] …
- [ ] …
```

## cheatsheet.md

Тільки команди, без пояснень. Розділи за сценаріями:

```text
Navigation | Files | Search | Network | SSH | Git | Python/Node | Homebrew | AI CLI | Danger Zone | Help
```

## 14-денний план (урок 10)

Формат кожного дня:

```text
День N: command1, command2, …
- Час: 20–30 хв
- Команд: 5–7 максимум
- Вправи: 3 практичні
- Контрольне питання: 1
```

## GIFT-квіз (опційно)

Формат Moodle GIFT на питання:

```text
::Question title::
Question text? {
  =Correct answer
  ~Incorrect answer 1
  ~Incorrect answer 2
  #### General feedback explaining the logic of the correct answer.
}
```

- 10 питань на розділ
- 1 правильна + 2 distractor'и
- Feedback лише для правильної відповіді
